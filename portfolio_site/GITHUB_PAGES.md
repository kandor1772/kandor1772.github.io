# Publish On GitHub Pages

Fastest path:

1. Create a new GitHub repository.
2. Upload everything in this folder:
   - `index.html`
   - `styles.css`
   - `config.js`
   - `.nojekyll`
3. In the repository:
   - open `Settings`
   - open `Pages`
   - under `Build and deployment`, choose:
     - `Deploy from a branch`
   - branch:
     - `main`
   - folder:
     - `/ (root)`
4. Save.
5. GitHub will give you a public site URL.

Recommended repo names:

- `portfolio`
- `tiberius-portfolio`
- `yourname.github.io`

Best option:

- if the repo is named `yourname.github.io`, the site becomes your main GitHub Pages homepage.

What to edit first:

1. Open `config.js`
2. Replace placeholder links
3. Update the card text

What this portfolio is good for:

- notebooks
- project summaries
- Tiberius live link
- Tiberius download links
- Larry project links

What it does not do by itself:

- run the Tiberius backend
- keep Tiberius live when your Mac is asleep
